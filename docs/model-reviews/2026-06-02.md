## 4. The change and its best-practice basis

**One focused improvement: extract the probability head into a pure, documented,
unit-testable function and lock its curve-shape invariants.**

Previously the head was inlined in `compute()`:

```rust
let prior_logodds = (HISTORICAL_ANCHOR.clamp(1e-9,0.5) / (1.0 - …)).ln();
let raw = sigmoid(prior_logodds + EVIDENCE_GAIN_SYS * l_sys).min(0.90);
```

so the formula could not be exercised in isolation, the four backtest point-anchors
pinned P at only four values of `L_sys`, and the `0.90` ceiling was a bare magic
literal. I extracted:

```rust
pub const P_CEILING: f64 = 0.90;
pub fn p_wwiii_from_l_sys(l_sys: f64) -> f64 { … }   // sigmoid(logit(baseline)+β·L_sys), capped
```

and call it from `compute()` (identical arithmetic — anchors unchanged). New tests in
`bayesian.rs` lock the **shape** of the head that the point-anchors leave free:

- `p_head_returns_flat_baseline_at_zero` — `L_sys = 0` returns the baseline exactly.
- `p_head_is_monotonic_nondecreasing_in_l_sys` — sweep `L_sys ∈ [0, 12]`, P never dips.
- `p_head_respects_engineering_ceiling` — saturates at, and never exceeds, `P_CEILING`.
- `p_head_reproduces_anchor_ladder` — the head alone, fed each analog's reported
  `L_sys`, lands in the right band (head locked independently of the theater path).

**Why this is the right, smallest change.** This directly implements the 2026-06-01
review's suggested next-step #3 ("a monotonicity/sanity-sweep test that walks `L_sys`
from 0 upward and asserts P is non-decreasing and bounded — cheap insurance against
future retunes"). It changes **no math** (anchors byte-identical) and so needs no
methodology edit, but it converts an implicit, untested contract into an explicit,
enforced one. This is standard proper-scoring-rule / calibration hygiene: the additive
log-odds combination form used here (the way Good-Judgment-Project- and Metaculus-style
aggregation fold evidence into a prior, rather than naïve probability multiplication)
is only well-behaved if it is monotone and bounded, and those properties should be
regression-locked, not assumed. Centralising the ceiling as `P_CEILING` also removes a
scattered magic literal — a named constant the head and any future caller now share.

## 6. Risks and uncertainties

- All anchors are **idealised synthetic analogs**, not live-corpus replays. The
  backtest header notes the live (fragile-ceasefire) corpus reads ~45% vs the idealised
  65% for the present world; I cannot verify the live number from here (no DB/logs).
- The duplicated `server.rs` test fix assumes the v2 "no dedicated theater-ladder strip"
  design is intentional — the committed `dashboard.html` comment says it is. If Robert
  actually wants a theater-ladder strip, the correct fix is to build it, not adjust the
  test (same caveat the 06-01 review raised).
- The new head tests lock the curve *shape*, not its *calibration values* — by design,
  so they survive a deliberate retune while still catching an accidental sign/clamp
  regression. The four-anchor band tests in `backtest.rs` remain the calibration locks.
- Tuning constants in `theater.rs` (`EVIDENCE_GAIN_SYS = 2.4`, the 0.45/0.30/0.12/0.70
  coupler multipliers, rung heat bands) are still fitted to four anchors only and are
  under-constrained between them.
